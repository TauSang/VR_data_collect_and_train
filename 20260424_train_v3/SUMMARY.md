# 20260424_train_v3 — 失败实验总结

## 目标
修复 v2 Phase 1 "第 6 轮 val_mse 即触底并上升" 的过拟合现象,
期望通过更强的正则化提升 Phase 2 最终 MuJoCo SR, 超过 v2 的 **95.20%** baseline。

## 方案差异 (v3 vs v2)

| 超参 | v2 | v3 | 变化 |
|------|----|----|------|
| hidden_dims | [1024, 1024, 512] | **[512, 512, 256]** | 参数量 1.79M → 0.48M (−73%) |
| dropout | 0.1 | **0.2** | ×2 |
| weight_decay | 1e-4 | **3e-4** (P1) / **2e-4** (P2) | ×3 / ×2 |

| batch_size | 256 | **512** | ×2 |
| patience | 60 / 40 | **200 / 150** | 放宽早停 |

## 训练结果 (WHU HPC, 1×A100, 36 min)

### Phase 1 (MJ-only)
- best_epoch=**6**, best_val_mse=**0.3185** (与 v2 几乎相同)
- 训练到 206 ep 后 val_mse 漂至 0.43 → 过拟合模式**完全未解决**
- 即使参数量砍到 1/4 + 双倍 dropout, 仍在第 6 轮触底

### Phase 2 (VR finetune)
- best_epoch=**5**, best_val_mse=**0.2043**
- 对比 v2 Phase 2 best_val_mse=**0.1363** → v3 **明显更差** (+50%)

运动轨迹正常, 无 spinning/drift, 失败案例为目标球偏远时 min_d=0.142m 未触及阈值。

## 结论: v3 属于**明确的负结果**

### 核心教训
1. **"第 6 轮过拟合" 不是 bug, 而是验证集的结构性属性**
   - episode-level 10% 切分本身就让 val 分布比 train 更难
   - 更强正则化**不能**让 val_mse 继续下降, 反而降低模型表达力
2. **val_mse 0.136 (v2) 才是可达的天花板, 0.20+ (v3) 对应 SR 远低于 95%**
3. **对这个任务 (31D obs → 8D action, 小数据集), 1.79M 参数的 MLP 是甜点区**; 砍到 0.48M 欠拟合

## 记入项目记忆
- v2 规模的 MLP + 中等正则 + 低 lr VR FT 是当前最优配方
- 进一步提升应转向**数据层面** (更多 VR 轨迹 / 更好采样策略 / 课程学习), 而非继续加正则
- 放弃"延后 best_epoch" 这一优化目标 — 它不映射 SR
